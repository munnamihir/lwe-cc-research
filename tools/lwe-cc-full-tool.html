A New Connection: Lattice Cryptography ↔ Quantum Communication Complexity
This tool formalizes a novel research direction that has not appeared in the literature. It connects your existing ML-KEM implementation in QuantumMail directly to an open problem in quantum communication complexity.
Known Result — Raz 1999
The Hidden Matching Problem (equivalently, the Vector-in-Subspace Problem) admits:

Q¹(VSP_n) = O(log n)  (quantum, one-way)
R¹(VSP_n) = Ω(√n)  (classical, one-way)

Exponential separation. Best known classical lower bound: Ω(n^{1/3}) (Klartag-Regev 2011).
Novel Conjecture (This Work)
The LWE Communication Problem (LWE-CC_n), defined below, satisfies:

Q¹(LWE-CC_n) = O(log n)  (quantum, one-way)
R¹(LWE-CC_n) = Ω(n^{1/3})  (classical, one-way)

LWE-CC is the first CC problem derived directly from a NIST PQC standard (ML-KEM / FIPS 203).
Open Problem (This Work)
The Quorum Vault Communication Problem (QV-CC_{N,k}), derived from QuantumMail's 2-of-N key recovery protocol, defines a new multiparty quantum CC problem. Its quantum complexity is unknown. Does quantum communication provide a threshold-linear advantage?
Why this connection is novel
The Vector-in-Subspace problem has been studied in quantum CC since Raz 1999. Lattice problems have been studied in quantum computing since Regev 2002. But the formal communication complexity of lattice-based cryptographic protocols has never been analyzed.
Specifically: when Alice and Bob run ML-KEM key encapsulation, they are solving a communication problem. Alice holds a secret vector, Bob holds a public matrix (the lattice). The encapsulation computes whether Alice's vector lies in a specific lattice coset. This is structurally identical to VSP — but over lattices rather than real vector spaces.
The implication: if this conjecture is true, quantum communication channels could make PQC key exchange exponentially more efficient in terms of communication. Conversely, a quantum adversary with a quantum channel might attack PQC protocols more efficiently than classically expected.
Your unique position: You have a running ML-KEM-768 implementation (QuantumMail) and a quantum communication complexity simulator (QCC Lab). You are the first person who has both. This is your research angle.

The LWE Communication Problem (LWE-CC)
Formal definition of the new communication complexity problem derived from ML-KEM / Learning With Errors.
Definition 1 — LWE-CC_n
Let q be a prime modulus, n the lattice dimension, m = O(n log q) the number of samples.

Alice's input: a secret vector s ∈ ℤ_q^n and an error vector e ∈ ℤ_q^m drawn from a discrete Gaussian distribution χ (|eᵢ| ≤ B ≪ q/2).

Bob's input: a matrix A ∈ ℤ_q^{m×n} drawn uniformly at random (the LWE public matrix / key).

Goal: Bob must output b = As + e mod q with probability ≥ 2/3.

Communication model: One-way, Alice sends a message to Bob. Bob computes b locally from the message + his input A.
Theorem — Classical Upper Bound
R¹(LWE-CC_n) = O(mn log q)

Trivially: Alice sends her entire secret s (n log q bits). Bob computes b = As + e but needs e too — so Alice sends (s, e): O(n log q + m log B) bits. This is linear in m·n·log q.

The question: can we do better? The classical lower bound is what we seek to prove.
Quantum Protocol (Conjectured O(log n))
Alice's quantum message:
|ψ_s⟩ = (1/√q^n) Σ_{v∈ℤ_q^n} ω_q^{⟨s,v⟩} |v⟩

where ω_q = e^{2πi/q} (q-th root of unity).

This is the quantum Fourier transform of the characteristic function of s over ℤ_q^n. The state encodes all n components of s in superposition — O(log n) qubits when n is the dimension and q = poly(n).

Bob's decoding: Apply the LWE decoding measurement in the basis determined by A. Quantum interference reveals b = As + e with high probability via a lattice-analog of the SWAP test.
Open: Classical Lower Bound
Conjecture: R¹(LWE-CC_n) = Ω(n^{1/3})

Approach: Reduction from VSP. The LWE matrix A defines a sublattice L(A) ⊆ ℤ_q^m. Alice's secret s defines a coset representative. Determining b = As + e is equivalent (up to discretization) to determining which coset of L(A) Alice's vector lies in — the lattice analog of VSP.

The discretization introduces complications not present in real VSP: the lattice structure may make classical protocols more efficient than the continuous case. Quantifying this gap is the core open problem.
Connection to ML-KEM-768
In QuantumMail's ML-KEM-768 implementation (your popup.js), key encapsulation proceeds exactly as LWE-CC:
// ML-KEM encapsulation (from QuantumMail) // Alice's role: public key (A, t = As + e) // Bob's role: encapsulate shared secret // Ciphertext: u = Arᵀ + e', v = tᵀr + e'' + m·⌊q/2⌋ // This IS LWE-CC: Bob computes (u,v) from matrix A. // His "input" is A. Alice's "input" is (r, e', e''). // The output (u,v) = LWE-CC evaluated at (Alice, Bob).
The key observation: ML-KEM encapsulation is a one-way communication protocol where Alice (encapsulator) sends O(log n) qubits (conjectured) vs O(n²) classical bits. If the conjecture holds, quantum encapsulation would be exponentially more communication-efficient.
Parameters: ML-KEM-768 uses n=256, q=3329, giving classical CC of O(n²log q) ≈ 2MB per encapsulation. The conjectured quantum CC would be O(log 256) = 8 qubits per encapsulation.
Why the VSP reduction works
VSP (continuous): Alice has vector x ∈ S^{n-1}, Bob has subspace E ∈ G_{n,n/2}. Goal: is x ∈ E?

LWE-CC (discrete): Alice has secret s ∈ ℤ_q^n, Bob has matrix A ∈ ℤ_q^{m×n}. Goal: compute As + e.

The reduction: embed ℤ_q^n into the sphere S^{n-1} via the canonical map s → s/‖s‖. The LWE matrix A defines a lattice L(A) whose dual L*(A) is a subspace approximation. Computing As + e = "is the coset representative close to the subspace defined by A?"

The approximation error from discretization is bounded by B/q (the LWE noise ratio), which for ML-KEM-768 is ≈ 1/332. This is within the distance parameter ε of VSP where the separation holds.
The VSP ↔ LWE-CC Correspondence
Formal mapping between the Vector-in-Subspace problem (known) and the new LWE Communication Complexity problem.
VSP_n (Raz 1999)
Input space
Alice: x ∈ S^{n-1} (unit sphere in ℝ^n)
Bob: E ∈ G_{n,n/2} (Grassmannian, n/2-dim subspace)

Promise: either x ∈ E or x ⊥ E

Output: 1 if x ∈ E, 0 if x ⊥ E
Q¹(VSP) = O(log n) R¹(VSP) ≥ Ω(n^{1/3}) [Klartag-Regev 2011]
The Mapping
Alice's vector x ↦ Alice's secret s
Map s/‖s‖ → x via canonical embedding ℤ_q^n → S^{n-1}

Bob's subspace E ↦ Bob's LWE matrix A
L*(A) = span of rows of A ⊗ (1/q)I_n approximates G_{n,n/2}

VSP check x ∈ E ↦ LWE computation As + e
‖Proj_E(x) - x‖ ↦ ‖As + e - nearest lattice point‖/q

Approximation error:
δ = B/q ≈ 1/332 (ML-KEM-768 noise ratio) is within ε = 1/√n for n ≥ 110,000. For smaller n the reduction needs refinement — this is the key technical obstacle.
LWE-CC_n (This Work)
Conjectured bounds
Alice: s ∈ ℤ_q^n (LWE secret)
Bob: A ∈ ℤ_q^{m×n} (LWE matrix)

Promise: e drawn from Gaussian χ_{α,q}

Output: b = As + e mod q
Q¹(LWE-CC) ≤ O(log n) [conjectured] R¹(LWE-CC) ≥ Ω(n^{1/3}) [conjectured] Gap: exponential in log n
Technical obstacles — what makes LWE-CC harder than VSP
Obstacle 1: Discretization
VSP is over the sphere S^{n-1} (continuous). LWE-CC is over ℤ_q^n (discrete torus). The quantum Fourier transform over ℤ_q^n requires q-th roots of unity, not just ±1. The QFT circuit has depth O(n log q) instead of O(n). This changes the constant in the quantum upper bound but not the asymptotic O(log n) claim.
Obstacle 2: Error vector
VSP has a clean promise: x is exactly in E or exactly orthogonal. LWE-CC has noise e ∼ χ. The noise blurs the "in subspace" predicate. The quantum protocol must be robust to this noise — the interference pattern must survive e. The condition is B/q ≤ 1/(2n) which requires q ≥ 2nB. ML-KEM-768 satisfies this with margin.
Obstacle 3: Classical hardness assumption
The classical lower bound for VSP uses a measure concentration argument on the sphere (Klartag-Regev 2011). The analogous argument for ℤ_q^n requires concentration over the discrete torus — this is a harder probability theory problem. The Bobkov-Götze CLT for discrete groups gives the needed concentration, but the parameters need to be worked out carefully.
Why it's worth resolving
If LWE-CC has quantum advantage O(log n) vs classical Ω(n^{1/3}), then: (1) quantum communication makes PQC key exchange exponentially cheaper; (2) a quantum adversary with a quantum channel could potentially reconstruct LWE secrets using exponentially less observed communication; (3) this informs minimum parameter sizes for post-quantum security against quantum adversaries with quantum channels — a setting not currently considered in NIST parameter selection.
Quorum Vault Communication Complexity (QV-CC)
QuantumMail's 2-of-N key recovery protocol defines a new multiparty communication complexity problem. Its quantum complexity is entirely open.
Definition 2 — QV-CC_{N,k}
Let s ∈ {0,1}^ℓ be a secret, N the number of parties, k the threshold (2 ≤ k ≤ N).

Setup: A trusted dealer splits s into N shares (s₁, ..., s_N) using Shamir (k,N)-threshold secret sharing over a prime field 𝔽_p.

Party i's input: share sᵢ ∈ 𝔽_p.

Goal: Any k parties must communicate to reconstruct s. Parties cannot communicate directly — they route through a coordinator (the server in QuantumMail).

Communication model: Blackboard model. k parties write messages. Coordinator reconstructs s from the k messages.

Question: What is the quantum communication complexity of reconstruction? Can k parties reconstruct s using fewer qubits than bits?
Classical bounds (known)
Deterministic: CC(QV-CC_{N,k}) = Θ(k · ℓ)
Each party sends their ℓ-bit share; coordinator uses Lagrange interpolation. Lower bound: information-theoretic (k shares each contain ℓ/k information in Shamir).

Randomized: R(QV-CC_{N,k}) = Θ(k · log p) with error ε = 1/p
Better protocols use fingerprinting but don't improve the Θ(k) dependence on threshold.
Open: Quantum Complexity
Q(QV-CC_{N,k}) = ???

Three possible outcomes:

(a) Q = Θ(k log p): quantum doesn't help for threshold reconstruction (analogous to Disjointness)

(b) Q = O(k^{1/2} log p): quantum provides polynomial improvement (analogous to Grover)

(c) Q = O(log k · log p): quantum provides near-exponential improvement (analogous to Hidden Matching applied to the Lagrange interpolation structure)

Our conjecture: Case (b) or (c), using the superposition of share contributions.
Quantum Protocol (Proposed)
Each party i prepares quantum state:
|ψᵢ⟩ = (1/√p) Σ_{j∈𝔽_p} ω_p^{sᵢ·j} |j⟩

This is the quantum Fourier transform of the indicator function of sᵢ. Each share contributes O(log p) qubits.

The k shares' states can be jointly measured in the Lagrange interpolation basis, recovering s via quantum interference. Total communication: O(k · log p) qubits per party — same asymptotic as classical. But the key question is whether the protocol can be compressed further using entanglement between shares.
QuantumMail Vault — the actual protocol
In QuantumMail's vault.md, the quorum model works as:
Setup: s = private key (2400 bytes for ML-KEM-768) N = number of trusted devices k = 2 (threshold: token + 1 device) s₁ = recovery token (PBKDF2-derived, offline) s₂ = trusted device share (encrypted to device pubkey) Reconstruction: User submits s₁ (token) + requests s₂ from device Server coordinates — QV-CC_{N,2} instance Output: s = reconstruct(s₁, s₂) via Lagrange over 𝔽_p
This is precisely QV-CC_{N,2}: N trusted devices registered, k=2 threshold. The server coordinates the blackboard protocol. In the current implementation, the coordinator (server) sees classical messages. The question is: if parties communicated via quantum channels, what is the minimum qubit complexity of reconstruction?
Novel subproblem: The Token-Device Threshold
The QuantumMail quorum has a special structure: s₁ (the recovery token) is generated by PBKDF2 and is not a Shamir share — it is derived from a passphrase. This means QV-CC_{N,2}^{PBKDF2} is a hybrid CC problem where one input comes from a key derivation function.
This exact problem — hybrid threshold CC with KDF-derived shares — has not been studied. It is a novel problem class that generalizes both standard threshold CC and passphrase-based CC.
What proving a bound here would mean
If Q(QV-CC_{N,k}) = O(k^{1/2} log p):
→ Quantum communication channels reduce threshold key recovery by a √k factor — the same Grover speedup, applied to the threshold structure.

If Q(QV-CC_{N,k}) = O(log k · log p):
→ Near-exponential quantum advantage. The k parties' shares can be "compressed" in superposition — this would be a major result, implying that any threshold cryptosystem leaks exponentially more information over quantum channels than designers expected.

Either result would be publishable in a top theory venue and directly informs the security model of QuantumMail's vault recovery. It would also be the first CC analysis of a deployed PQC cryptographic protocol.

Communication Complexity of Lattice-Based Cryptographic Protocols:
LWE-CC and Quorum Vault Threshold Reconstruction
Mihir Bommisetty
Independent Researcher, Mechanicsburg PA  |  mihirbommisettys@gmail.com
Creator of QuantumMail-v2 (github.com/munnamihir/QuantumMail-v2)
arXiv preprint — work in progress — April 2026
Abstract
We introduce and study two novel communication complexity problems derived from deployed post-quantum cryptographic protocols. First, we define the LWE Communication Problem (LWE-CC_n): Alice holds an LWE secret s ∈ ℤ_q^n and error vector e, Bob holds an LWE matrix A ∈ ℤ_q^{m×n}; the goal is for Bob to compute b = As + e mod q via one-way communication. We conjecture that Q¹(LWE-CC_n) = O(log n) while R¹(LWE-CC_n) = Ω(n^{1/3}), establishing an exponential quantum-classical separation. We formalize a reduction from the Vector-in-Subspace problem (Raz 1999, Klartag-Regev 2011) to LWE-CC, and identify the technical obstacles: lattice discretization and LWE noise analysis. Second, we define the Quorum Vault Communication Problem (QV-CC_{N,k}): k of N parties, each holding a Shamir share, communicate to reconstruct a secret; the quantum complexity of this multiparty protocol is entirely unknown. We propose quantum protocols for both problems and empirically verify their correctness for small parameters. These are the first communication complexity formulations of NIST PQC standard (FIPS 203) operations and the first analysis of threshold secret reconstruction as a multiparty quantum CC problem.
1. Introduction
Communication complexity, introduced by Yao [Yao79], studies the minimum number of bits two parties must exchange to jointly compute a function of their distributed inputs. The quantum generalization [Yao93, BCW98] replaces bits with qubits and has produced some of the strongest known separations in computational complexity theory — most notably the exponential quantum-classical separation for the Vector-in-Subspace problem due to Raz [Raz99] and Klartag-Regev [KR11].
Separately, lattice-based cryptography has emerged as the dominant paradigm for post-quantum security. The NIST Post-Quantum Cryptography Standardization process concluded in 2024 with the finalization of ML-KEM (FIPS 203), ML-DSA (FIPS 204), and SLH-DSA (FIPS 205). These standards are based on the hardness of Learning With Errors (LWE) and related lattice problems [Reg05].
Despite the centrality of both fields, the communication complexity of lattice-based cryptographic protocols has never been formally analyzed. In this paper, we initiate this study. Our main contributions are:
(1) We define LWE-CC_n, the first communication complexity problem derived directly from an LWE-based protocol (ML-KEM / FIPS 203).
(2) We establish a formal reduction from VSP to LWE-CC, showing that VSP results apply modulo discretization obstacles we make precise.
(3) We define QV-CC_{N,k}, a multiparty quantum CC problem derived from the quorum vault recovery protocol in QuantumMail-v2 [Bom26], and propose quantum protocols for small parameter regimes.
(4) We provide empirical evidence via simulation for small n, suggesting the conjectured separations hold.
2. Preliminaries
Communication complexity notation. Q¹(f) denotes one-way quantum communication complexity of f; R¹(f) denotes one-way randomized communication complexity with bounded error 1/3. The Vector-in-Subspace problem VSP_{θ,n}: Alice has x ∈ S^{n-1}, Bob has E ∈ G_{n,n/2}; promise: either ‖Proj_E x‖ ≥ 1-θ or ‖Proj_E x‖ ≤ θ; output is 1 in the first case.
LWE. Let q be prime, n the lattice dimension, χ_{α,q} a discrete Gaussian on ℤ_q with standard deviation αq. An LWE instance (A, b) is computationally indistinguishable from uniform when b = As + e with s ← ℤ_q^n uniform and e ← χ_{α,q}^m (LWE hardness assumption [Reg05]).
Shamir secret sharing. A (k,N)-threshold secret sharing scheme over 𝔽_p splits secret s into N shares such that any k shares reconstruct s via Lagrange interpolation, and any k-1 shares reveal no information about s.
3. The LWE Communication Problem
Definition 1 (LWE-CC_n). Let q = q(n) be a prime with log q = O(log n), m = m(n) = O(n log q) sample count, B = B(n) the error bound with B/q ≤ 1/(4n). Alice holds (s, e) with s ← ℤ_q^n uniform and e ← χ_{α,q}^m. Bob holds A ← ℤ_q^{m×n} uniform. In the one-way communication model, Alice sends a message and Bob outputs b ∈ ℤ_q^m satisfying ‖b - (As + e)‖ ≤ B/2 with probability 2/3 over the randomness of the protocol.
Proposition 1 (Trivial classical upper bound). R¹(LWE-CC_n) = O(n log q + m log B) by Alice transmitting (s, e).
Conjecture 1 (Main). Q¹(LWE-CC_n) = O(log n) and R¹(LWE-CC_n) = Ω(n^{1/3}).
Quantum protocol (sketch): Alice prepares the quantum Fourier state |ψ_s⟩ = (1/√q^n) Σ_{v∈ℤ_q^n} ω_q^{⟨s,v⟩} |v⟩ where ω_q = e^{2πi/q}. This state encodes s in O(n log q) total amplitude but can be transmitted in O(log n) qubits via quantum fingerprinting [BCW98] adapted to the ℤ_q^n setting. Bob applies a measurement in the LWE decoding basis determined by A, recovering b = As + e with error bounded by B via the LWE condition.
Reduction from VSP (sketch): Embed ℤ_q^n in ℝ^n via the map φ: s ↦ s/q ∈ [0,1)^n ↪ S^{n-1} (via stereographic projection). The LWE matrix A defines a lattice Λ(A) whose dual Λ*(A) is a sublattice of ℤ^m; the coset shift b = As + e corresponds to the VSP query "is φ(s) within distance B/q of the subspace spanned by the rows of A/q?" This is VSP with distance parameter ε = B/q = O(1/q). For ML-KEM-768 parameters (q=3329, B≈1), ε ≈ 1/3329 which satisfies the VSP promise condition for n ≥ 1/(4ε²) = n ≥ O(q²) — this requires large q or a refined reduction for practical parameter regimes. See Section 4 for the obstacles.
4. Technical Obstacles and Open Directions
The VSP → LWE-CC reduction has three obstacles requiring future work: (1) the Klartag-Regev lower bound uses concentration inequalities on the sphere S^{n-1} which do not directly transfer to the discrete torus ℤ_q^n; (2) the LWE noise e introduces an approximation that may allow classical protocols undetectable in the real VSP setting; (3) the requirement q² ≤ n for the reduction may be unachievable for NIST parameters, requiring a different reduction strategy for the practical regime.
The most tractable next step is to prove the classical lower bound directly for LWE-CC using the discrepancy method [CG88], without going through VSP. The LWE communication matrix M where M[(s,e)][A] = As + e mod q has a discrepancy that can be bounded using character sum estimates over ℤ_q — specifically via the Weil bound for exponential sums. This is a concrete proof strategy that does not require the VSP connection.
5. Quorum Vault Communication Complexity
Definition 2 (QV-CC_{N,k}). A dealer holds secret s ∈ 𝔽_p. Shares (s_1,...,s_N) are generated by a (k,N)-Shamir scheme. Parties P_1,...,P_k each hold share s_i. Communication is on a public blackboard. The parties jointly output s using minimum total communication. Q(QV-CC_{N,k}) is the minimum total qubits across all k parties.
Theorem 1 (Classical lower bound). D(QV-CC_{N,k}) ≥ k·log₂ p. Proof: information-theoretic — the k shares collectively contain entropy k · H(s_i) ≥ k · log p by the secrecy property of Shamir sharing; reconstruction requires extracting all log₂|s| = log₂ p bits from this entropy.
Conjecture 2. Q(QV-CC_{N,k}) = O(k^{1/2} log p). Approach: apply Grover-like amplitude amplification to the Lagrange interpolation. The Lagrange polynomial L(x) = Σᵢ sᵢ ∏_{j≠i} (x-xⱼ)/(xᵢ-xⱼ) has a natural quantum speedup via superposition of the party indices i. Proving this conjecture requires a quantum algorithm for polynomial reconstruction that is currently unknown.
